## 注意事项

- 默认情况下，用户在登录前需要验证其电子邮件地址。要关闭这个功能，请在[你的项目](https://cloud.memfiredb.com)中禁用**确认电子邮件**。
- **确认电子邮件**决定了用户在注册后是否需要确认他们的电子邮件地址。
  - 如果**确认电子邮件**被启用，将返回一个`用户`，但`会话`为空。
  - 如果**确认电子邮件**被禁用，则同时返回一个`用户`和一个`会话`。
- 当用户确认他们的电子邮件地址时，他们默认会被重定向到`SITE_URL`。你可以修改你的`SITE_URL`或在[你的项目](https://cloud.memfiredb.com)中添加额外的重定向URL。
- 如果为一个现有的确认用户调用signUp()。
    - 如果在[你的项目](https://cloud.memfiredb.com)中启用了**确认邮件**，将返回一个混淆的/假的用户对象。
    - 如果**确认电子邮件**被禁用，将返回错误信息 `用户已注册`。
- 要获取当前登录的用户，请参考[`getUser()`](/docs/reference/javascript/auth-getuser)。










## 实例

### Sign up.



```js
const { data, error } = await supabase.auth.signUp({
  email: 'example@email.com',
  password: 'example-password',
})
```

### 用附加的用户元数据进行注册。



```js
const { data, error } = await supabase.auth.signUp(
  {
    email: 'example@email.com',
    password: 'example-password',
    options: {
      data: {
        first_name: 'John',
        age: 27,
      }
    }
  }
)
```
